Fahh Editor docs

A Tauri 2 desktop IDE built on Rust, React 18, and Monaco. Full file tree, integrated terminal, optional AI tools — and one feature that no other IDE has.

🔊

The fahhhh.mp3 filename must never change. The file lives at src-tauri/assets/fahhhh.mp3 (four h's). Renaming it breaks the core feature. This is a hard rule.

Fahh Editor is currently at v0.1.0 (Phase 1). The Phase 1 scope covers the full editor shell: file tree, Monaco editor, integrated terminal, Fahh SFX, and the optional tools installer. LSP integration, the Git sidebar, and the AI panel are Phase 2.

Windows 10/11 macOS 11+ Linux (GTK3)

System overview

Fahh Editor uses a two-process model. The Rust backend owns the OS surface; the React frontend owns the UI. They communicate via Tauri IPC.

┌─────────────────────────────────────────────────────────────┐ │ Frontend — WebView2 (Windows) / WebKit (macOS, Linux) │ │ React 18 · TypeScript strict · Monaco · Zustand · Tailwind │ │ │ │ ┌──────────┐ ┌────────────────┐ ┌──────────────────┐ │ │ │ FileTree │ │ EditorPane │ │ TerminalPanel │ │ │ │ sidebar │ │ (Monaco) │ │ (cmd runner) │ │ │ └──────────┘ └────────────────┘ └──────────────────┘ │ │ ↑ invoke() / emit() │ ├────────────────────────┼──────────────────────────────────────┤ │ Rust backend — Tauri 2 core │ │ ┌─────────────┐ ┌─────────────┐ ┌──────────────────┐ │ │ │ workspace │ │ terminal │ │ error_detector │ │ │ │ (fs ops) │ │ (exec cmd) │ │ (fahh SFX) │ │ │ └─────────────┘ └─────────────┘ └──────────────────┘ │ │ Plugins: tauri-plugin-shell · tauri-plugin-dialog · fs │ └─────────────────────────────────────────────────────────────┘ ↓ OS syscalls File system · Processes · LSP servers

All Tauri commands are registered in src-tauri/src/lib.rs via .invoke_handler(tauri::generate_handler![...]). The frontend calls them with invoke() from @tauri-apps/api/core. Every command has a corresponding typed wrapper in src/lib/tauri.ts.

// architecture

Data flow

Opening a file

User clicks file in FileTree → FileNode.handleClick() → useWorkspace.openFileInEditor(path) → tauri.ts: readFile(path) → invoke("read_file", {path}) → Rust: workspace::read_file() → std::fs::read_to_string() → editorStore.openFile(doc, content) → EditorPane re-renders with Monaco showing content

Saving a file

Ctrl+S in EditorPane → EditorPane.handleSave() → tauri.ts: writeFile(path, content) → invoke("write_file", {path, content}) → Rust: workspace::write_file() → std::fs::write() → editorStore.markDirty(path, false) → TabBar dirty indicator (●) clears

Terminal command

User types command, presses Enter → useTerminal.run(command) → tauri.ts: executeCommand(command, [], cwd) → invoke("execute_command", {command, args, cwd}) → Rust: terminal::execute_command() → std::process::Command("cmd /C ...") → Returns stdout + stderr + exit_code → terminalStore.addLine(stdout) → Also emits "terminal://output" event (for future streaming)

Fahh SFX trigger

Rust: error_detector::ErrorDetector::trigger(app) → Loads atomic timestamp, checks cooldown (default 3 s) → If cooldown elapsed: app.emit("fahh://error", ()) → JS: fahh.ts listen("fahh://error", handler) → Web Audio API: AudioContext → BufferSource → destination.start() → User hears fahhhh.mp3

Events

These events are emitted by Rust and received by the frontend via listen() from @tauri-apps/api/event.

fahh://error
error_detector.rs
Triggers fahhhh.mp3 playback. Payload: null. Respects the configured cooldown.
terminal://output
terminal.rs
Emitted after each command completes. Payload: {stdout, stderr, exit_code}.
installer://progress
installer.rs
Streams install progress. Payload: {tool, status: "starting" | "done" | "error", message}.
// fahh sfx

How the sound works

The Fahh SFX is the core feature. It plays fahhhh.mp3 when errors are detected. The filename has four h's. Do not rename it.

Trigger sources

In Phase 1, the trigger must be called manually from Rust code. Phase 2 will wire LSP diagnostic callbacks and build output parsing to call it automatically.

rust
let detector = ErrorDetector::new(3); // 3-second cooldown
detector.trigger(&app_handle);          // emits fahh://error if cooldown passed

Frontend playback

Call initFahhSfx() once on app mount. It loads the MP3 via fetch(), decodes it into an AudioBuffer, and registers the Tauri event listener. Tear it down on unmount.

typescript
import { initFahhSfx, teardownFahhSfx } from "./lib/fahh";

// In App.tsx:
useEffect(() => {
  initFahhSfx();
  return () => teardownFahhSfx();
}, []);
// fahh sfx

SFX configuration

SettingDefaultRangeDescription
sfx_cooldown_secs30 – 30Minimum seconds between fahh plays. 0 = no limit.

Set via saveConfig({ sfx_cooldown_secs: 5, ... }) from the frontend, or edit ~/.fahh/config.json directly.

⚠️

Setting cooldown to 0 with a noisy LSP (many diagnostics per keystroke) will play the sound extremely frequently. Start at 3 seconds.

// configuration

config.json schema

The config file lives at ~/.fahh/config.json and is created automatically on first run.

json
{
  "sfx_cooldown_secs": 3,          // seconds between fahh plays
  "last_workspace": "/path/to/dir", // reopened on next launch (Phase 2)
  "installed_tools": ["gh", "n8n"], // tools installed via wizard
  "theme": null                       // reserved for Phase 2
}

All fields are optional. Missing fields use defaults. The file is safe to edit by hand — it's read once on startup and written via save_config.

// optional tools

Installer wizard

The optional tools installer runs everything locally with no Docker. Open it via the ⚙ button in the activity bar.

ToolRequiresInstall command
n8nNode 18+npm install -g n8n
browser-usePython 3.11+pip install browser-use
Flowise AINode 18+npm install -g flowise
GitHub CLI—winget install GitHub.cli
Claude CodeNode 18+npm install -g @anthropic-ai/claude-code

The installer calls get_tool_status() on open to check which tools are already on PATH, then calls install_tool(name) for each install request. Progress is streamed via installer://progress events.

ℹ️

The installer is idempotent — running it twice is safe. It always checks for an existing install before running the install command.
